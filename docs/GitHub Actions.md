# GitHub Actions (깃허브 액션)

> "내 코드를 푸시하면 알아서 움직이는 나만의 24시간 비서"

GitHub에서 제공하는 [[CI/CD]] 및 워크플로우 자동화 도구다. 별도의 서버 설치 없이 깃허브 저장소(Repository) 안에서 모든 것이 해결된다는 게 최대 장점이다.

## 핵심 개념들

- **Workflow**: 자동화된 절차 전체. `.github/workflows/*.yml` 파일로 정의한다.
- **Event**: "누가 푸시했을 때", "PR을 날렸을 때" 처럼 이 비서를 깨우는 신호.
- **Job**: 독립된 환경(컨테이너)에서 실행되는 작업 단위.
- **Step**: Job 안에서 순차적으로 실행되는 개별 명령(커맨드).
- **Action**: 미리 만들어진 재사용 가능한 부품. "로그인하기", "빌드하기" 등의 조각들.

## 왜 좋은가?

1. **관리 포인트 제로**: [[Jenkins]]처럼 직접 서버를 구축하고 업데이트할 필요가 없다. (SaaS 형태)
2. **GitHub 연동**: PR이 날아오면 자동으로 테스트를 돌리고 결과를 댓글로 달아주는 등의 연동이 환상적이다.
3. **무료 (Public)**: 공개 저장소는 무제한 무료다. 개인 프로젝트에는 이만한 게 없다.

## 사용 예시 (YAML)

```yaml
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Test
        run: npm test
```

## 실제 활용

- **자동 린트/테스트**: 코드가 규칙에 맞는지, 버그는 없는지 항상 체크.
- **자동 배포**: 마스터 브랜치에 코드가 합쳐지면 [[AWS]]나 [[Docker]] Hub로 자동 배포.
- **알림**: 작업이 끝나면 디스코드나 슬랙으로 메시지 전송.

## 기타

1. **Marketplace**: 남들이 만들어둔 액션이 수만 개다. 웬만한 작업은 검색해서 복사/붙여넣기만 하면 된다.
2. **Self-hosted Runner**: 보안상의 이유로 외부 서버를 쓰기 싫다면, 내 서버를 직접 깃허브 액션 일꾼(Runner)으로 등록해 쓸 수도 있다.
3. **Secrets**: API 키나 비밀번호는 유출되지 않게 'Secrets' 설정에 따로 보관해야 한다.
