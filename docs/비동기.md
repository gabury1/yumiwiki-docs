# 비동기 (Asynchronous)

> 기다리지 않고 다음 일을 처리하는 효율성의 미학

작업을 요청한 후 결과가 나올 때까지 기다리지 않고(Blocking 되지 않고), **다음 작업을 즉시 수행**하는 방식. 결과 처리는 나중에 완료 신호(Callback, Future, Promise)가 오면 진행한다. 웹 서버, UI 프로그래밍, [[네트워크]] 통신에서 필수적인 개념이다.

## 왜 필요한가?

**Blocking (블로킹)**: 커피 주문을 하고 커피가 나올 때까지 계산대 앞에서 멍하니 기다리는 것. 뒤 사람들은 주문도 못 한다. (비효율적)
**Non-Blocking (논블로킹)**: 진동벨을 받고 자리에 가서 폰을 하다가, 벨이 울리면 커피를 받아오는 것. (효율적)

컴퓨터에서 CPU는 엄청나게 빠르고, I/O(하드디스크 읽기, 네트워크 요청)는 엄청나게 느리다. 비동기를 쓰지 않으면 느린 I/O를 기다리느라 빠른 CPU가 놀게 된다.

## 언어별 구현

### [[JavaScript]] / [[TypeScript]]
비동기가 언어의 핵심이다. 싱글 스레드지만 이벤트 루프([[Event Loop]])를 통해 비동기를 처리한다.
- 과거: Callback Hell (콜백 지옥)
- 현재: `Promise`, `async/await` 문법 사용.

### [[Java]] / [[Spring Boot]]
- 전통적으로는 멀티 스레드(Thread Pool)로 해결했다. (요청당 스레드 하나)
- [[Spring]] 5부터 **WebFlux**가 도입되면서 리액티브 프로그래밍(Reactor)을 통한 완전한 비동기/논블로킹을 지원한다.
- Java 21+에서는 **Virtual Threads**가 도입되어, 동기 코드처럼 짜면서도 비동기의 효율을 낼 수 있게 되었다.

### [[Python]]
- `asyncio` 라이브러리와 `async/def` 문법을 통해 단일 스레드 동시성을 지원한다. [[FastAPI]]가 이를 적극 활용한다.

## 관련 개념

- **동시성(Concurrency)**: 여러 작업이 번갈아가며 진행되어 동시에 하는 것처럼 보이는 것. (논리적)
- **병렬성(Parallelism)**: 물리적으로 CPU 코어 여러 개가 동시에 작업하는 것.
- **Callback**: 일이 끝나면 호출해달라고 넘겨주는 함수.
- **Future / Promise**: "지금은 없지만 나중에 값을 주겠다"는 약속 객체.

## 주의점

**디버깅이 어렵다**: 실행 순서가 뒤죽박죽이라 스택 트레이스 추적이 어렵다.
**코드 복잡도**: 콜백이 꼬이거나 상태 관리가 잘못되면 "Race Condition(경쟁 상태)"이 발생할 수 있다.
