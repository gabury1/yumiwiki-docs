# 보안

> 시스템과 데이터를 안전하게 지켜야 할 때 필요한 개념

허가되지 않은 접근, 사용, 공개, 파괴로부터 시스템과 정보를 보호하는 것이다. 집에 자물쇠 달고, 금고에 귀중품 넣고, CCTV 설치하는 것과 같다.

## 탄생 배경

컴퓨터가 네트워크로 연결되면서 외부 공격이 가능해졌다. 해커들이 데이터를 훔치고, 시스템을 망가뜨리고, 돈을 요구했다. 개인정보 유출, 서비스 마비 같은 사고가 반복되면서 보안의 중요성이 커졌다.

## 용도

금융 서비스에서 고객 정보와 거래 데이터를 보호한다. 의료 시스템에서 환자 기록을 안전하게 관리한다. 모든 웹 서비스에서 [[인증]], [[인가]]를 통해 접근을 제어한다.

사용자 비밀번호를 해시로 저장하고, HTTPS로 통신을 암호화하고, 방화벽으로 외부 공격을 막는다.

## 핵심 원칙

**기밀성(Confidentiality)**: 허가된 사람만 정보에 접근.

**무결성(Integrity)**: 정보가 변조되지 않음을 보장.

**가용성(Availability)**: 필요할 때 정보에 접근 가능.

이 세 가지를 CIA 삼각형이라고 부른다.

## 생태계

- [[인증]] - 누구인지 확인 ([[OAuth2]], [[JWT]])
- [[인가]] - 무엇을 할 수 있는지 결정
- [[암호화]] - 데이터를 읽을 수 없게 변환
- [[방화벽]] - 네트워크 경계 보호
- [[WAF]] - 웹 애플리케이션 공격 방어
- [[OWASP Top 10]] - 주요 웹 취약점 목록

## 기타

"보안은 귀찮은 것"이라는 말이 있다. 편의성과 보안은 항상 트레이드오프 관계다. 비밀번호를 복잡하게 만들면 안전하지만 기억하기 어렵다. 이중 인증 켜면 안전하지만 로그인할 때마다 번거롭다. 완벽한 보안과 편의성을 동시에 가질 수는 없다.

"완벽한 보안은 없다"는 것도 업계의 공통된 인식이다. 시스템을 완전히 안전하게 만드는 건 불가능하고, 비용과 위험을 고려해서 적절한 수준을 정해야 한다. 은행 앱과 개인 블로그의 보안 수준이 같을 필요는 없다.

해커들이 쓰는 "사회공학(Social Engineering)" 공격은 기술이 아니라 사람을 속이는 방식이다. 아무리 시스템을 튼튼하게 만들어도, 직원이 피싱 메일에 넘어가서 비밀번호를 알려주면 끝이다. "보안의 가장 약한 고리는 사람"이라는 말이 있다.

"Zero Trust" 모델이 요즘 트렌드다. "사내망이니까 안전하다"는 가정을 버리고, 내부든 외부든 모든 접근을 검증한다는 개념. 재택근무가 늘면서 경계 기반 보안의 한계가 드러났고, Zero Trust가 대안으로 떠올랐다.
