# Tailwind CSS (테일윈드)

> "CSS 파일을 아예 만들지 않아도 웹사이트를 완성할 수 있는 마법"

**Utility-First**를 지향하는 CSS 프레임워크다. `container.css` 같은 별도의 스타일 시트를 만드는 대신, HTML 클래스에 이미 정의된 속성(`flex`, `pt-4`, `text-blue-500`)을 때려넣는 방식으로 개발한다.

## 왜 쓰는가? (비유)

- **전통 방식(Vanilla CSS)**: 옷을 하나하나 재단해서 만드는 맞춤복. (깔끔하지만 오래 걸림)
- **부트스트랩(Bootstrap)**: 이미 완성된 옷 세트. (빠르지만 남들이랑 똑같음)
- **테일윈드**: 완벽한 질의 수만 가지 옷 조각들. (내 마음대로 조립하는데 엄청 빠름)

## 특징

**개발 속도 대폭발**: HTML과 CSS 파일을 왔다 갔다 할 필요가 없다. 클래스 이름 짓느라(`header-inner-left-box-v2`...) 고통받지 않아도 된다.

**일관된 디자인 시스템**: `p-4`는 항상 16px이다. 개발자들끼리 눈대중으로 하던 간격 맞추기가 시스템적으로 정리된다.

**최종 용량이 작다**: 빌드 시점에 안 쓰는 클래스는 다 버린다(Purge). 결국 내가 쓴 만큼만 CSS 파일이 생성된다.

## 사용 예시

```html
<div class="flex items-center space-x-4 p-6 bg-white rounded-xl shadow-lg">
  <div class="text-xl font-medium text-black">안녕하세요</div>
  <p class="text-slate-500">테일윈드 세상입니다.</p>
</div>
```

## 호불호 (논란)

- **호**: "한 번 써보면 다신 예전으로 못 돌아간다." 생산성이 미쳤다.
- **불호**: "HTML이 너무 더러워진다." 가독성이 떨어지고 구조 파악이 어렵다는 비판이 있다.

## 기타

1. **JIT (Just-In-Time) 엔진**: 예전엔 모든 클래스를 다 로드해서 무거웠는데, 이제는 입력하는 순간 즉석에서 CSS를 생성한다.
2. **반응형 디자인**: `md:flex`, `lg:grid` 처럼 클래스 앞에 접두사만 붙이면 반응형 대응이 끝난다. 미디어 쿼리 안녕!
3. **Headless UI**: 테일윈드 제작진이 만든 디자인 없는 컴포넌트 라이브러리다. 테일윈드와 찰떡궁합이다.
