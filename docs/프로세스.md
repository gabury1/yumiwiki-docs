# 프로세스 (Process)

> "하드디스크에 잠자던 '프로그램'이 메모리에 올라가 살아 움직이기 시작하는 순간"

컴퓨터에서 실행 중인 프로그램의 인스턴스다. 운영체제로부터 자원을 할당받는 작업의 단위이며, 독립적인 실행 환경을 가진다.

## 메모리 구조 (독방 시스템)

프로세스는 운영체제로부터 자신만의 독립된 [[메모리]] 영역을 할당받는다. 이 영역은 크게 4가지로 나뉜다.

1.  **Code (Text)**: 작성한 소스 코드가 기계어로 저장되는 영역. (읽기 전용)
2.  **Data**: 전역 변수와 정적 변수가 저장되는 영역.
3.  **[[힙]](Heap)**: 개발자가 동적으로 할당하는 영역. (자유로운 공간)
4.  **[[스택]](Stack)**: 함수 호출 시 생성되는 지역 변수와 매개변수가 저장되는 영역. (생겼다 금방 사라지는 공간)

## 특징

-   **독립성**: 프로세스 A는 프로세스 B의 메모리에 접근할 수 없다. (접근하려면 IPC라는 복잡한 통신이 필요함)
-   **자원 점유**: CPU 시간, 메모리, 파일 핸들 등을 운영체제로부터 직접 할당받는다.
-   **제어 블록(PCB)**: 운영체제가 프로세스를 관리하기 위해 만드는 신분증 같은 데이터 구조다.

## 프로세스의 상태

프로세스는 일생 동안 상태가 계속 변한다.
-   **New**: 이제 막 생성됨.
-   **Ready**: 메모리에 올라갔고, CPU가 불러주기만을 기다림.
-   **Running**: 실제로 CPU를 점유해서 실행 중.
-   **Waiting**: 입출력(I/O) 작업을 기다리느라 잠시 쉬는 중.
-   **Terminated**: 실행 종료.

## 기타

1.  **멀티 프로세싱**: 여러 개의 프로세스가 동시에 돌아가는 것. 하나가 죽어도 다른 프로세스에 영향을 주지 않아 안정적이지만, 자원을 많이 먹는다. (예: 구글 크롬의 탭들)
