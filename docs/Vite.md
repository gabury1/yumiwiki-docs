# Vite

> 빠른 프론트엔드 개발 환경이 필요할 때

차세대 프론트엔드 빌드 도구다. [[Vue]] 창시자 Evan You가 만들었다. 2020년에 나왔고, [[Webpack]]을 대체하고 있다.

개발 서버가 엄청 빠르다. ES 모듈 네이티브 사용해서 번들링 없이 바로 실행. HMR도 밀리초 단위.

## 성능

개발 시 번들링 안 함. 브라우저가 import 요청하면 그때 변환. 프로젝트 커져도 시작 속도 일정.

프로덕션 빌드는 [[Rollup]] 사용. 최적화된 번들 생성.

HMR이 빠름. 파일 수정하면 거의 즉시 반영. [[Webpack]] 대비 체감 10배 이상.

## 생태계

[[React]], [[Vue]], [[Svelte]], [[Solid]] 다 지원. 공식 템플릿 있음.

플러그인 시스템이 Rollup과 호환. 필요한 건 대부분 있음.

## 주의점

**IE 미지원**: 모던 브라우저만. 레거시 필요하면 `@vitejs/plugin-legacy`.

**CJS 의존성 문제**: CommonJS 패키지 호환 이슈 가끔 있음. 대부분 해결되지만 가끔 삽질.

**Webpack 플러그인 호환 안 됨**: Webpack 생태계 그대로 못 씀.

## 주요 기능

- 즉시 시작 개발 서버
- 네이티브 ES 모듈
- 빠른 HMR
- Rollup 기반 프로덕션 빌드
- 다양한 프레임워크 지원
- TypeScript 기본 지원

## 기타

[[Vue]] 창시자 Evan You가 만들었다. 2020년 공개됐는데, 불과 몇 년 만에 [[Webpack]]의 아성을 위협하고 있다. 프랑스어로 "빠르다"는 뜻. 실제로 체감 속도가 압도적이어서 이름값한다.

"No Bundle" 개발 서버가 혁신이었다. [[Webpack]]은 개발 서버 띄울 때도 전체를 번들링하는데, Vite는 요청이 올 때 그때그때 변환한다. 브라우저의 네이티브 ES 모듈을 활용한 덕분. 프로젝트가 커져도 시작 속도가 일정하다.

Vite 3부터는 Vue뿐 아니라 [[React]], [[Svelte]], [[Solid]] 등 다양한 프레임워크를 지원한다. 공식 템플릿도 제공한다. "Vue 전용"이라는 인식을 깨고 범용 빌드 도구로 자리잡았다.

[[Astro]], [[SvelteKit]], [[Nuxt 3]] 같은 메타 프레임워크들이 Vite를 내부적으로 쓴다. 빌드 도구 레이어에서 사실상 표준이 되고 있다. "2020년대 빌드 도구 = Vite"라는 공식이 형성되는 중.
