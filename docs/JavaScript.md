# JavaScript

> 웹 브라우저에서 돌아가는 유일한 언어

1995년 Brendan Eich가 Netscape에서 10일 만에 만든 언어다. 원래 이름은 Mocha, 그다음 LiveScript, 마케팅 목적으로 Java 인기에 편승해 JavaScript가 됐다. [[Java]]와는 전혀 다른 언어다.

처음엔 브라우저에서 간단한 인터랙션용이었다. 2009년 [[Node.js]]가 나오면서 서버에서도 쓸 수 있게 됐다. 현재는 웹 프론트엔드, 백엔드, 모바일([[React Native]]), 데스크톱([[Electron]]) 어디서든 쓰인다. 세계에서 가장 많이 쓰이는 프로그래밍 언어 중 하나.

## 탄생 배경

1990년대 중반, 웹 페이지는 정적인 HTML뿐이었다. Netscape는 브라우저에서 동적인 기능을 원했다. Brendan Eich에게 "10일 안에 프로토타입 만들어"라고 했고, 정말 10일 만에 나왔다. 급하게 만들어서 이상한 부분이 많다. `typeof null === 'object'`가 대표적인 실수.

Microsoft가 JScript로 따라하면서 브라우저 호환성 전쟁이 시작됐다. ECMA에서 표준화해서 ECMAScript가 됐다. ES5(2009), ES6/ES2015가 주요 전환점. ES6에서 `let`, `const`, 화살표 함수, 클래스, Promise가 추가됐다.

## 특징

**동적 타이핑**: 변수 타입 선언 안 해도 된다. `let x = 5; x = "hello";` 가능. 유연하지만 런타임 에러 원인이 된다. [[TypeScript]]가 나온 이유.

**프로토타입 기반**: 클래스 대신 프로토타입 체인으로 상속. ES6에서 `class` 문법 추가됐지만 내부는 여전히 프로토타입.

**일급 함수**: 함수를 변수에 담고, 인자로 넘기고, 반환할 수 있다. 콜백 패턴, 고차 함수의 기반.

**비동기 논블로킹**: [[이벤트 루프]]와 콜백 큐로 비동기 처리. 싱글 스레드지만 I/O 작업을 효율적으로 처리한다. Promise, async/await로 콜백 지옥 해결.

**느슨한 타입 변환**: `"5" + 3 = "53"`, `"5" - 3 = 2`. 암묵적 타입 변환이 혼란을 준다. `===` 쓰는 이유.

## 성능

V8 엔진([[Chrome]], [[Node.js]])이 JIT 컴파일로 꽤 빠르다. 순수 연산은 [[C]]보다 느리지만, I/O 바운드 작업에서 효율적이다. [[이벤트 루프]] 덕분에 동시 요청 처리에 강하다.

[[Node.js]]는 싱글 스레드라서 CPU 바운드 작업에 약하다. Worker Threads로 우회 가능하지만 복잡하다. 무거운 연산은 다른 언어나 WebAssembly 고려.

## 생태계

**패키지 관리**: [[npm]] (세계 최대 패키지 저장소), yarn, pnpm

**프론트엔드 프레임워크**: [[React]], [[Vue]], [[Angular]], [[Svelte]]

**백엔드**: [[Node.js]], [[Express]], [[NestJS]], [[Fastify]]

**빌드 도구**: [[Webpack]], [[Vite]], [[esbuild]], [[Rollup]]

**테스트**: [[Jest]], [[Mocha]], [[Vitest]], Cypress

**모바일**: React Native, Ionic

**데스크톱**: Electron, Tauri

## 주의점

**this 바인딩**: 호출 방식에 따라 `this`가 달라진다. 화살표 함수는 렉시컬 `this`. 이거 때문에 버그 많이 난다.

**비동기 처리**: 콜백 지옥, Promise 체인, async/await 섞이면 혼란. 에러 핸들링 빼먹기 쉽다.

**브라우저 호환성**: 구형 브라우저 지원하려면 [[Babel]] 트랜스파일 필요. 폴리필도 챙겨야 한다.

**타입 없음**: [[TypeScript]] 안 쓰면 대규모 프로젝트에서 유지보수 힘들다.

**== vs ===**: `==`는 타입 강제 변환. `===` 습관화 필수.

**`null` vs `undefined`**: 둘 다 "없음"인데 미묘하게 다르다. 일관성 있게 쓰는 게 중요.

## 주요 기능

- [[이벤트 루프]] 기반 비동기 처리
- Promise, async/await
- 화살표 함수, 스프레드 연산자, 구조 분해
- 모듈 시스템 (ES Modules, CommonJS)
- 프로토타입 체인 상속
- 클로저, 고차 함수

## 기타

JavaScript와 Java는 "햄"과 "햄스터"만큼 관련 없다. 이름만 비슷하고 문법, 패러다임, 용도 다 다르다. Netscape가 Java 인기에 편승하려고 이름을 바꾼 거다. 이 때문에 수십 년째 오해받는다.

Brendan Eich는 10일 만에 언어를 만들었다는 걸 계속 후회한다고 했다. 급하게 만들어서 이상한 부분이 많은데, 하위 호환성 때문에 고칠 수 없다. `typeof null === 'object'`는 초기 구현 버그인데, 고치면 기존 코드가 깨져서 영원히 남게 됐다.

"JavaScript fatigue"라는 말이 있다. 프레임워크, 빌드 도구, 라이브러리가 너무 빨리 바뀌어서 개발자들이 지친다는 뜻. 2016년쯤 가장 심했고, 요즘은 좀 안정됐다. 그래도 매년 새로운 프레임워크가 나온다.

Stack Overflow 설문에서 11년 연속 "가장 많이 쓰이는 언어" 1위. 웹 개발의 유일한 선택지라서 그런 면도 있다. 좋든 싫든 피할 수 없는 언어다.
