# C

> 시스템 프로그래밍과 하드웨어 제어가 필요할 때

1972년 Dennis Ritchie가 Bell Labs에서 만든 언어다. Unix 운영체제를 다시 작성하기 위해 개발됐다. "이식 가능한 어셈블리"라고 불린다. 하드웨어에 가깝게 접근하면서도 어셈블리보다 생산성이 높다.

운영체제, 임베디드 시스템, 드라이버, 게임 엔진의 핵심 언어다. Linux, Windows, macOS 커널이 C로 작성됐다. [[Python]], [[Ruby]], [[PHP]] 인터프리터도 C로 만들어졌다. 프로그래밍 언어의 "어머니".

## 탄생 배경

1960년대 후반, Bell Labs에서 Unix를 개발 중이었다. 어셈블리로 OS 짜는 건 힘들었다. Ken Thompson이 B 언어를 만들었고, Dennis Ritchie가 이를 개선해 C를 만들었다.

1973년 Unix가 C로 다시 작성됐다. OS를 고급 언어로 쓴 최초 사례. 이후 Unix가 다양한 하드웨어에 이식되면서 C도 퍼졌다. 1978년 "The C Programming Language" (K&R) 책이 나왔고, 사실상 표준이 됐다.

## 특징

**저수준 접근**: 포인터로 메모리 직접 조작. 하드웨어 레지스터 접근 가능. 어셈블리 인라인 삽입도 가능.

**수동 메모리 관리**: `malloc`, `free`로 직접 할당/해제. 가비지 컬렉션 없음. 효율적이지만 메모리 릭, 댕글링 포인터 위험.

**절차적 프로그래밍**: 함수 중심. 객체지향 아님. 구조체(struct)로 데이터 묶지만 메서드는 없다.

**작은 언어**: 키워드 32개. 문법 단순. 표준 라이브러리도 최소한. 배우기 쉽지만 직접 할 게 많다.

**이식성**: 어셈블리보다 이식 가능. 컴파일러만 있으면 다른 아키텍처에서 돌린다.

## 성능

컴파일 언어 중 가장 빠른 축에 속한다. 어셈블리 바로 위 수준. 오버헤드가 거의 없다.

최적화 컴파일러(GCC, Clang)가 매우 발달해 있다. 인간이 짠 어셈블리보다 빠를 때도 있다.

하지만 최적화는 개발자 책임이다. 잘못 짜면 느리다. 메모리 레이아웃, 캐시 친화적 코드, 분기 예측 등 저수준 지식 필요.

## 버전

**K&R C (1978)**: "The C Programming Language" 책 기준. 함수 선언 방식이 현대와 다름.

**ANSI C / C89 (1989)**: 표준화. 함수 프로토타입, void 타입 등.

**C99 (1999)**: 인라인 함수, 가변 길이 배열, `//` 주석, `long long`.

**C11 (2011)**: 멀티스레딩, `_Generic`, 익명 구조체.

**C17 (2017)**: 버그 수정 위주.

**C23 (2023)**: `nullptr`, `typeof`, `constexpr` 등 현대화.

## 생태계

**컴파일러**: GCC (GNU), Clang (LLVM), MSVC (Windows)

**빌드 도구**: Make, CMake, Meson, Ninja

**디버거**: GDB, LLDB, Valgrind (메모리 검사)

**표준 라이브러리**: libc (glibc, musl, uclibc)

**주요 사용처**: Linux 커널, Git, SQLite, Redis, Python/Ruby 인터프리터

## 주의점

**메모리 안전성**: 버퍼 오버플로우, use-after-free, 메모리 릭. 보안 취약점의 주범. 조심히 다뤄야 한다.

**미정의 동작(UB)**: 컴파일러가 예측 불가능한 동작을 할 수 있다. `int a[10]; a[11];` 같은 코드.

**포인터 복잡성**: 이중 포인터, 함수 포인터 등 어렵다. 버그 원인 1순위.

**문자열 처리**: null-terminated 문자열. `strlen`, `strcpy` 버퍼 오버플로우 위험. `strncpy` 쓰자.

**에러 처리**: 예외 없음. 반환값으로 에러 전달. `errno` 전역 변수. goto로 cleanup 패턴.

## 주요 기능

- 포인터와 포인터 연산
- 구조체(struct), 공용체(union)
- 전처리기(#define, #include, #ifdef)
- 표준 입출력(stdio.h)
- 동적 메모리 할당(stdlib.h)
- 인라인 어셈블리

## 기타

Dennis Ritchie와 Ken Thompson은 2011년 컴퓨터 분야 최고 영예인 튜링상을 공동 수상했다. Unix와 C의 공로. Ritchie는 같은 해 세상을 떠났는데, Steve Jobs 사망 일주일 후라 주목받지 못했다는 아쉬움이 있다.

"C는 총이다. 발등을 쏠 자유가 있다"라는 말이 있다. 강력한 만큼 위험하다는 뜻. [[Rust]]가 "메모리 안전한 C"를 표방하며 등장한 이유다.

K&R (Kernighan & Ritchie)의 "The C Programming Language"는 프로그래밍 서적의 고전이다. "Hello, World" 예제가 여기서 시작됐다. 얇지만 내용이 밀도 있어서 지금도 추천된다.

"C를 알면 컴퓨터가 어떻게 동작하는지 안다"라는 말이 있다. 메모리, 포인터, 시스템 콜 등 저수준 개념을 이해하게 된다. CS 전공에서 필수로 가르치는 이유다.
