# Go

> 단순하고 빠른 서버 사이드 개발이 필요할 때

2009년 [[Google]]이 만든 언어다. Robert Griesemer, Rob Pike, Ken Thompson([[C]] 공동 창시자)이 설계했다. "복잡해진 [[C++]]에 대한 반발"로 시작됐다. 단순함, 빠른 컴파일, 동시성 처리가 특징.

[[Docker]], [[Kubernetes]], Terraform, Hugo 등 클라우드 인프라 도구들이 Go로 만들어졌다. 컨테이너, [[마이크로서비스]] 시대의 시스템 프로그래밍 언어로 자리잡았다.

## 탄생 배경

2007년 Google에서 시작됐다. C++ 컴파일이 45분 걸리는 동안 "더 나은 언어를 만들자"고 논의했다는 일화가 있다.

목표는 명확했다. C++의 성능, [[Python]]의 생산성, 대규모 시스템에서의 유지보수성. 모든 기능을 넣는 대신, 최소한의 핵심 기능만 제공하기로 했다.

2009년 오픈소스 공개. 2012년 Go 1.0 출시. 하위 호환성 약속. Go 1으로 작성한 코드는 Go 1.x에서 계속 돌아간다.

## 특징

**단순함**: 키워드 25개. 제네릭(Go 1.18까지 없었음), 예외, 상속 없음. 배우기 쉽고 읽기 쉬움.

**빠른 컴파일**: 대규모 프로젝트도 수 초 내 컴파일. 의존성 관리가 효율적.

**정적 타입 + 타입 추론**: 타입 안전하면서도 `:=`로 간결하게 선언.

**가비지 컬렉션**: 자동 메모리 관리. 저지연 GC 계속 개선 중.

**Goroutine**: 경량 스레드. 수십만 개 동시 실행 가능. OS 스레드보다 훨씬 가볍다.

**Channel**: Goroutine 간 통신. "메모리 공유 대신 통신으로 공유하라" CSP 모델.

**단일 바이너리**: 의존성 다 포함해서 하나의 실행 파일로 컴파일. 배포 간단.

## 성능

컴파일 언어라 빠르다. [[Java]]보다 빠르고, [[C]]보다는 느리지만 근접.

GC가 있어서 [[C]]/[[C++]]만큼의 극한 성능은 안 나온다. 하지만 대부분의 서버 애플리케이션에서 충분.

Goroutine 덕분에 동시성 처리가 효율적. 수만 연결 처리하는 서버를 쉽게 만들 수 있다.

## 생태계

**웹 프레임워크**: Gin, Echo, Fiber (미니멀), Buffalo (풀스택)

**CLI 도구**: Cobra, urfave/cli

**ORM**: GORM, ent

**테스트**: 표준 `testing` 패키지, testify

**주요 프로젝트**: [[Docker]], [[Kubernetes]], Terraform, Prometheus, InfluxDB, CockroachDB

## 주의점

**에러 처리**: 예외 없이 반환값으로 처리. `if err != nil` 반복이 많다. 보일러플레이트 논란.

**제네릭**: 1.18에서야 추가됐다. 이전엔 `interface{}`와 타입 단언으로 우회. 아직 제네릭 생태계가 성숙 중.

**의존성 관리 역사**: 초기엔 GOPATH 지옥. Go Modules(1.11+)로 해결됐지만 아직 레거시 프로젝트 있다.

**GUI 앱**: 웹, CLI, 서버에 강하지만 GUI 앱 개발 생태계는 약하다.

**표현력**: 단순함의 대가로 표현력이 제한적. 메타프로그래밍, 매크로 없음.

## 주요 기능

- Goroutine (경량 동시성)
- Channel (CSP 기반 통신)
- Interface (덕 타이핑)
- defer (함수 종료 시 실행)
- 다중 반환값
- 내장 테스트, 벤치마크, 프로파일링
- go fmt (코드 포맷팅 표준화)

## 기타

Go의 마스코트는 Gopher(땅다람쥐)다. Renée French가 디자인했다. 귀여운 캐릭터 덕분에 Gopher 관련 굿즈가 많다. Go 개발자를 "Gopher"라고 부른다.

"Go는 21세기의 C다"라는 말이 있다. 시스템 프로그래밍에 적합하면서도 C보다 안전하고 생산적이다. 클라우드 인프라 도구들이 Go로 만들어진 이유.

코드 포맷이 강제다. `go fmt`이 표준 포맷 적용. 탭 vs 스페이스 논쟁 없음. 팀 코드 스타일 통일이 자동으로 된다.

Google에서 만들었지만 Kubernetes 덕분에 CNCF 생태계의 사실상 표준 언어가 됐다. 클라우드 네이티브 개발자에게 Go는 필수 스킬이 됐다.
